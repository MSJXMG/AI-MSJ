
# 📜 GitHub 开源协议选择指南 (精简版)

### 🚀 快速决策表
| 你的目标 | 推荐协议 | 一句话特点 |
| :--- | :--- | :--- |
| **极致简单** | **MIT** | 只要保留我的名字，你干啥都行。 |
| **商业/专利保护** | **Apache 2.0** | 适合大项目，防止他人滥用专利反告你。 |
| **强迫开源 (传染性)** | **GPL v3** | 你用了我的，你的项目也必须开源。 |
| **放弃版权** | **Unlicense** | 完全公共领域，像垃圾一样扔掉。 |

---

### 🔍 核心协议大白话

#### 1. MIT License（最常用）
* **属性：** 宽容、极简。
* **规则：** 必须在代码中保留原作者的版权声明。
* **适用：** 个人小工具、简历项目、前端库（如 Vue/React 插件）。

#### 2. Apache License 2.0（最稳妥）
* **属性：** 严谨、商业友好。
* **规则：** 除了保留声明，还提供了明确的专利授权。
* **适用：** 企业级框架、涉及复杂算法的项目、担心专利纠纷的项目。

#### 3. GNU GPLv3（最硬核）
* **属性：** 理想主义、传染性。
* **规则：** 任何修改或使用该代码的派生项目，**都必须**以 GPL 协议开源。
* **适用：** 坚定的开源支持者，防止代码被闭源商用。

---

### 🛠️ 建议操作流程
1.  **新建仓库**：在 GitHub 创建页面勾选 **Add a license**。
2.  **默认首选**：如果不确定选什么，直接选 **MIT**。
3.`Agent` 技能或 `skillsCLI` 等工具，推荐 **MIT** 以获得最大传播率，或 **Apache 2.0** 以保护架构逻辑。

> **注意：** 一旦选定并发布，虽然可以更改，但会对之前的版本产生复杂的法律溯及问题。建议“首选 MIT，求稳选 Apache”。
